# D-008 — Транзитний рахунок ПриватБанку як спеціальний випадок

**Дата:** 2026-05-16

**Рішення:** Список ЄДРПОУ транзитних рахунків зберігається в налаштуваннях (`settings.transit_edrpou_list`). Якщо платник по такому рахунку — пропускаємо двофакторну перевірку і покладаємось тільки на номер договору з призначення.

**Альтернативи:**

- Не виокремлювати транзит, обробляти всі платежі однаково.
- Fuzzy-матч по ПІБ з призначення після коми.

**Обґрунтування:** В аналізі виписки 35 платежів (7.4%) приходять через транзитний рахунок ПриватБанку (ЄДРПОУ 14360570). Реальний платник — фізособа, чий ПІБ вказаний після коми в призначенні. ЄДРПОУ платника в таких випадках — це ЄДРПОУ банку, і він не може бути використаний для матчингу. Якщо номер договору в призначенні є, цього достатньо для ідентифікації.

**Наслідки:**

- В налаштуваннях окреме поле `transit_edrpou_list`.
- Логіка класифікації має спеціальну гілку для транзитних платежів.
- Fuzzy-матч по ПІБ не реалізуємо в MVP (опційно як підказка для адміна в адмінці пізніше).
