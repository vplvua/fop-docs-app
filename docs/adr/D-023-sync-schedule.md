# D-023 — Sync Моє ОСББ: enum-розклад (first/last/manual) + ручна кнопка

**Дата:** 2026-05-17

**Рішення:**

1. Автоматичний sync з MySQL "Моє ОСББ" виконується **раз на місяць** замість попереднього "раз на день" (D-004).
2. Розклад керується новим ключем `Settings.moeosbb_sync_schedule` (enum: `first` | `last` | `manual`):
   - `first` — sync 1-го числа місяця. **Default.**
   - `last` — sync в останній день місяця.
   - `manual` — без автоматичного, тільки за ручною дією адміна.
3. В UI розділу 8.7 (Налаштування) — кнопка "Синхронізувати зараз", доступна завжди незалежно від `moeosbb_sync_schedule`.

**Альтернативи:**

- Cron-вираз замість enum. Відкинуто: гнучкіше, але складніший UI; для соло-адмін-сценарію 3 опції достатньо.
- Залишити "раз на день" (D-004). Відкинуто: дані клієнтів в "Моє ОСББ" змінюються рідко (раз на місяць — максимум); щоденний sync — це навантаження на БД "Моє ОСББ" без бізнес-цінності.
- Default = `last` замість `first`. Відкинуто: `first` синхронізує дані ПЕРЕД потенційними платежами на початку місяця; нові клієнти в "Моє ОСББ" встигнуть потрапити в нашу БД до того, як від них прийде платіж.
- Додати telemetry-поле `Settings.moeosbb_last_sync_at`. Відкинуто з Settings: це ad-hoc стан системи, не конфігурація — зберігати в окремій runtime-таблиці або обчислювати з логу sync-операцій.

**Обґрунтування:**

- Користувач (власник "Моє ОСББ") вказав, що раз на місяць достатньо — дані клієнтів змінюються нечасто.
- Ручна кнопка покриває потребу "щойно додав клієнта в Моє ОСББ — хочу його одразу в нашу БД".
- Enum простіший і безпечніший за cron (немає невалідних виразів, які не виконаються тихо).

**Наслідки:**

- В `Settings` додається ключ `moeosbb_sync_schedule` (enum, default `first`).
- В розділі моделі "MySQL Моє ОСББ (read-only)" формулювання "раз на день" замінюється на "за розкладом з `settings.moeosbb_sync_schedule`".
- В PRD розділі 7.3 (Моє ОСББ): підпункт "Періодичність sync" уточнюється з посиланням на enum.
- В PRD розділі 8.7 (Налаштування) додається підпункт "Кнопка ручної синхронізації".
- Версія `domain_model.md` → 0.7 (мінор: новий Settings-ключ + зміна sync-логіки).

**Переглядає:** D-004 (там було "sync раз на день"; тут уточнюється на enum-розклад з default = монтлі).
