# D-030 — Алерти про проблеми системи: банер у адмінці, без push-каналу

**Дата:** 2026-05-18

**Рішення:**

1. У MVP алерти про збої зовнішніх API і fail класифікації показуються **тільки як банер у адмінці** (вгорі дашборду 8.1 і відповідного розділу).
2. Push-канали (Telegram-бот, email, SMS) **не реалізуються в MVP**. Можуть бути додані в Phase 2 (PRD 11) після оцінки реального профілю інцидентів.
3. Дані для банера обчислюються з логу останніх API-викликів і не зберігаються окремо: останній статус-код, ts останньої успішної взаємодії, тип помилки.
4. Контент банера: ім'я сервісу + код/тип проблеми + час останньої успішної операції. Приклад: `"ПриватБанк API: 401 Unauthorized, остання успішна синхронізація 4 год тому"`.

**Альтернативи:**

- **Telegram-бот** (реюз з privatbank-telegram-bot). Відкинуто на MVP: для single-admin сценарію, де власник заходить у адмінку як мінімум щоденно і має кнопку "Синхронізувати зараз" (D-023) — push дублює функцію без істотного виграшу. Якщо інциденти стануть частими — додамо в Phase 2 з контекстом, як часто адмін реально пропускає прихід платежів.
- **Email на адресу власника.** Відкинуто: ризик потрапити в spam, ризик пропустити в перевантаженому inbox, потреба налаштовувати SMTP. Дублює UI-індикатор без додаткової вартості для single-admin.

**Обґрунтування:**

- Single-admin (D-001), нечасті інциденти (PrivatBank і Дубідок — стабільні сервіси), polling-архітектура з ретраями автоматично переживає короткі збої.
- Кожен push-канал додає інфраструктури (SMTP, Telegram bot, secrets, тестування доставки). На MVP не виправдано.
- UI 8.7 уже планує "статус підключень" як read-only індикатор (D-020) — банер аналогічний механізм, тільки виноситься на дашборд для видимості.

**Наслідки:**

- В PRD 8.1 (Інформаційна архітектура) додається ділянка "Стан системи" з банерами.
- В PRD 10.4 (Логування і моніторинг) фіксується: алерти = UI-банер; push не реалізується в MVP.
- В моделі — без змін. Останні API-виклики логуються в окрему runtime-таблицю (поза скоупом domain model — це integration log, не бізнес-сутність).
