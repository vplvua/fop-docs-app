# D-007 — Один платіж = один акт (без агрегації)

**Дата:** 2026-05-16

**Рішення:** На кожен платіж генерується окремий акт. Не агрегуємо платежі за місяць.

**Альтернативи:**

- Накопичувати платежі клієнта за місяць і генерувати один акт у кінці місяця.

**Обґрунтування:**

- Власник як ФОП на 3 групі ЄП виставляє акти **по факту оплати**, не по факту надання послуги.
- Для клієнтів ОСББ зручніше, коли акт відповідає платіжному документу.
- Якщо в одному місяці прийшло 2 платежі по 200 грн (наприклад, доплата за минулий місяць + наперед за наступний) — це 2 окремих акти, що відповідає реальній бізнес-логіці.
- Платежі за різні типи послуг (доступ + СМС) однаково мають бути окремими актами через різні шаблони.

**Наслідки:**

- Cardinality `Payment` ↔ `Act` = 1:1.
- Акт може бути згенерований одразу після класифікації платежу (не чекати кінця місяця).
- Дата акту все одно = останній день місяця платежу (юридична вимога).
- Платежі, які мали б породити кілька актів (наприклад, 480 грн = 200 за доступ + 280 за СМС), не агрегуються автоматично — попадають у чергу для ручного розбору.
