# D-013 — Сторінка тарифів одразу, не як Phase 2

**Дата:** 2026-05-16

**Рішення:** В адмінці з самого старту є повноцінна сторінка тарифної сітки з CRUD правил. На старті в сітці одне правило: `apartments_min=0, apartments_max=NULL, price=200`.

**Альтернативи:**

- Зашити константу 200 в код + поле `access_price_override` на клієнті. Сітку додати в Phase 2, коли реально знадобиться.

**Обґрунтування:** Інфраструктура тарифу через сітку не сильно складніша за hardcoded константу, а перехід "константа → сітка" пізніше потребуватиме міграції. Власник хоче готовність до диференціації цін за розміром ОСББ.

**Наслідки:**

- Доменна модель з самого старту має `Tariff` як окрему сутність.
- Логіка резолву ціни підтримує сітку від першого дня.
