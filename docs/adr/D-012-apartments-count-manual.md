# D-012 — apartments_count заповнюється вручну в адмінці, не з MySQL

**Дата:** 2026-05-16

**Рішення:** Поле `Client.apartments_count` заповнюється тільки вручну в адмінці. Sync з "Моє ОСББ" його не торкається.

**Альтернативи:**

- Витягувати з "Моє ОСББ" автоматично через sync.

**Обґрунтування:** В "Моє ОСББ" дані про apartments_count можуть бути неповними/неточними. Власник готовий заповнити цю інформацію вручну для всіх ~300 клієнтів одноразово. Це також знімає залежність моделі тарифікації від схеми даних у "Моє ОСББ" — якщо завтра в "Моє ОСББ" зміниться структура, тарифи не зламаються.

**Наслідки:**

- В міграції на старті потрібен крок одноразового заповнення apartments_count.
- Sync-логіка простіша (немає мапінгу складних полів).
