# D-003 — Список виключень для нестандартних клієнтів

**Дата:** 2026-05-16

**Рішення:** Поле `Client.auto_act_disabled` (bool). Якщо true — платежі цього клієнта класифікуються (попадають у чергу з reason=auto_act_disabled), але акт не генерується автоматично. Адмін обробляє вручну.

**Альтернативи:**

- Не виокремлювати, обробляти автоматично (з ризиком неправильних актів).
- Окремий статус "ignore_payments" для контрагентів, чиї платежі не наша справа.

**Обґрунтування:** В аналізі виписки виявлено 3-4 контрагенти (ФОП Урбан, Авалон, Портмоне, Континент) з нестандартними договорами, де автогенерація може дати неправильний результат. Власник готовий обробляти їх вручну. Окремий `ignore_payments` визнаний надмірним — простіше пропускати ці платежі в адмінці після перегляду.

**Наслідки:**

- Прапорець `auto_act_disabled` на клієнті.
- Окрема enum-причина в черзі.
- В адмінці видно ці платежі окремо, з кнопкою "пропустити" або "згенерувати акт вручну з параметрами".
