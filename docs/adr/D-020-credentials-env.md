# D-020 — Credentials виносяться з Settings у env

**Дата:** 2026-05-17

**Рішення:**

1. З таблиці `Settings` видаляються три ключі: `privatbank_token`, `dubidoc_token`, `moeosbb_db_credentials`.
2. Ці credentials зберігаються виключно в environment variables процесу:
   - `PRIVATBANK_TOKEN`
   - `DUBIDOC_TOKEN`
   - `MOEOSBB_DB_URL` — одна connection string у форматі `mysql://user:pass@host:port/dbname`
3. В `Settings` лишаються тільки бізнес-параметри, якими адмін реально керує з UI: ціни/quantity SMS, polling interval, transit EDRPOU list, regex-патерни, SMS keywords.
4. UI розділу 8.7 (Налаштування) НЕ показує токени і НЕ дозволяє їх редагувати. Натомість показує **read-only статус підключення** до кожного зовнішнього сервісу (наприклад, "✓ Дубідок: остання операція 5 хв тому" / "✗ ПриватБанк: 401 на останньому запиті"), що обчислюється з логу останніх API-викликів.

**Альтернативи:**

- Лишити в Settings з encryption (як зараз). Відкинуто: дамп БД виносить токени; шифрування з ключем у тому ж процесі = ілюзія захисту; з ключем окремо = додаткова операційна складність ради функції, яка дає env.
- Гібрид: токени в env, але "remind in DB" (енкриптована копія для UI). Відкинуто як надмірне.
- Окремий secret store (Vault, AWS Secrets Manager). Відкинуто: overkill для соло-проекту на маленькій VPS.
- Окремі MOEOSBB_DB_HOST/USER/PASSWORD/... змінні замість одної URL. Відкинуто: більше шансів пропустити одну при копіюванні між середовищами; стандартний драйвер MySQL приймає URL.

**Обґрунтування:**

- 12-factor app: secrets — це config, config — це env.
- Дамп БД (бекап, дебаг) безпечно передавати — не несе credentials.
- Ротація токену = зміна env + рестарт процесу. Простіше і безпечніше за форму в UI з валідацією і аудитом.
- Для соло-адмін-сценарію немає бізнес-need керувати credentials через UI.

**Наслідки:**

- В моделі `Settings` видаляються рядки `privatbank_token`, `dubidoc_token`, `moeosbb_db_credentials`.
- Створюється `.env.example` (на етапі коду) з переліком обов'язкових змінних.
- PRD розділ 8.7: видаляється підпункт "Токени API і креди"; додається підпункт "Статус підключень до зовнішніх сервісів".
- PRD розділ 10.3 (Безпека): фіксується env-only підхід; описуються вимоги (не комітити .env, не логувати значення env, не відображати в UI).
- CLAUDE.md: додається правило "secrets — тільки env, ніколи в Settings".
- Версія `domain_model.md` → 0.6.1 (patch — видалення рядків з config-таблиці, не зміна логіки).
