# D-002 — Не використовуємо інвойси

**Дата:** 2026-05-16

**Рішення:** Система не виставляє інвойси/рахунки. Працюємо тільки з парою (договір → платіж → акт).

**Альтернативи:**

- Виставляти інвойси автоматично першого числа місяця і матчити платежі на інвойси.

**Обґрунтування:**

- Поточна бізнес-модель власника не використовує інвойси (юридично достатньо договору, де вказано оплату на розрахунковий рахунок).
- Виставлення інвойсів неможливо змінити в існуючому процесі.
- Більшість платежів і так мають у призначенні номер договору, що дає матчинг.
- Інвойси додавали б шар складності без виграшу для власника.

**Наслідки:**

- Цикл "очікуваний платіж → нагадування → прострочка" поза скоупом.
- Матчинг будується на парсингу призначення + ЄДРПОУ платника, не на ID інвойсу.
