# D-005 — Нумерація актів окрема для кожного клієнта

**Дата:** 2026-05-16

**Рішення:** Нумерація актів — окрема для кожного клієнта. Формат `№M` для першого акту в місяці M, `№M/N` для додаткових. Приклад: `№5`, `№5/2`, `№5/3`.

**Альтернативи:**

- Глобальна наскрізна нумерація по всіх клієнтах.
- Нумерація по договору з нескінченним лічильником.

**Обґрунтування:** Поточна практика автора. Просто, відповідає очікуванням клієнтів і їхній бухгалтерії. Не потребує глобального counter-а і race-condition-safe (унікальність гарантується по `(client_id, year, month, sub_number)`).

**Наслідки:**

- В БД унікальний індекс на `(client_id, act_date)` за форматом року-місяця.
- Простий алгоритм генерації наступного номера.
