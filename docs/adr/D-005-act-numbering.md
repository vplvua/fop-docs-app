# D-005 — Нумерація актів окрема для кожного клієнта

**Дата:** 2026-05-16 (формат оновлено 2026-05-30, зміна `match-act-to-sample`)

**Рішення:** Нумерація актів — окрема для кожного клієнта. Формат `MM/YYYY` для першого акту в місяці M року Y, `MM/YYYY/N` для додаткових (N — порядковий номер). Місяць — з провідним нулем. Приклад: `04/2026`, `04/2026/2`, `04/2026/3`.

> Попередній формат (до 2026-05-30) був `№M` / `№M/N` (напр. `№4`, `№4/2`). Змінено на `MM/YYYY[/N]` для повної відповідності паперовим зразкам актів (`docs/samples/acts/act-2026-04_200.pdf`).

**Альтернативи:**

- Глобальна наскрізна нумерація по всіх клієнтах.
- Нумерація по договору з нескінченним лічильником.
- Старий формат `№M` без року.

**Обґрунтування:** Поточна практика автора та вигляд раніше виданих клієнтам актів. Просто, відповідає очікуванням клієнтів і їхній бухгалтерії. Не потребує глобального counter-а і race-condition-safe (унікальність гарантується по `(client_id, act_date, number)`).

**Наслідки:**

- В БД унікальний індекс на `(client_id, act_date, number)`.
- Простий алгоритм генерації наступного номера (`formatActNumber` у `lib/acts/numbering.ts`, продубльований у `generateActNumber` у `lib/classification/act-stub.ts` — тримати в синхроні).
- Рік береться з `act_date` (останній день місяця платежу, ADR D-006).
