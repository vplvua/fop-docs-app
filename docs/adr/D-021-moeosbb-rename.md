# D-021 — Перейменування mosbb*\* → moeosbb*\*; mosbb_client_id → moeosbb_user_id

**Дата:** 2026-05-17

**Рішення:** Системне виправлення транслітерації + узгодження з реальною схемою БД "Моє ОСББ":

1. Префікс `mosbb_` (типо — пропущена `e` від "Моє") замінюється на `moeosbb_` у всіх ідентифікаторах коду і конфігурації.
2. Поле `Client.mosbb_client_id` водночас перейменовується на `Client.moeosbb_user_id` — щоб назва відображала реальну таблицю в БД "Моє ОСББ" (вона називається `osbb_users`, не `osbb_clients`).
3. Env-змінні з D-020 також правляться: `MOSBB_DB_URL` → `MOEOSBB_DB_URL`.

**Альтернативи:**

- Лишити `mosbb_` як було. Відкинуто: typo буде розмножуватись по коду, env-конфігах і документації.
- Перейменувати тільки `mosbb_client_id` без правки typo в інших місцях. Відкинуто: створить неконсистентність всередині однієї логічної групи (поля і env-змінні, що стосуються однієї зовнішньої системи).
- Перейменувати на `mo_osbb_` (з підкресленням) чи інший варіант. Відкинуто: `moeosbb_` ближче до природної транслітерації і не плутає читача.

**Обґрунтування:**

- "Моє ОСББ" → у нормальній транслітерації `Moe OSBB` → у snake_case `moeosbb`. Без `e` префікс читається як "MOSBB" — щось зовсім інше.
- Користувач (власник проекту і "Моє ОСББ") явно вказав, що `mosbb` — це орфографічна помилка, не свідома абревіатура.
- Назва `moeosbb_user_id` водночас вирішує другу проблему — таблиця в БД називається `osbb_users`, тому FK-поле логічно називати `*_user_id`.

**Наслідки:**

- В моделі: `Client.mosbb_client_id` → `Client.moeosbb_user_id`.
- В sync-правилах: "sync працює тільки для клієнтів з `moeosbb_user_id IS NOT NULL`".
- В описі `Contract.number`: "Стандартно дорівнює `moeosbb_user_id`".
- В env-змінних: `MOSBB_DB_URL` → `MOEOSBB_DB_URL` (D-020 актуалізується).
- В історичних згадках decisions_log (D-004, D-011, D-019, D-020) і CHANGELOG ([0.6.1]) текст оновлюється з технічним перейменуванням; суть рішень не торкається.
- В PRD_skeleton розділ 3.5 — формулювання sync переписується з новою назвою.
- В CLAUDE.md — без змін (CLAUDE.md не містить згадок mosbb\_\*).
- Версія `domain_model.md` → 0.6.2 (patch — косметичне rename без зміни логіки).
