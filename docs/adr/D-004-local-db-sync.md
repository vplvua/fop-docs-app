# D-004 — Локальна БД, sync з MySQL "Моє ОСББ" тільки на читання

**Дата:** 2026-05-16

**Рішення:** Окрема локальна БД для системи бухгалтерії. Sync з MySQL "Моє ОСББ" — read-only, періодичний. Snapshot реквізитів — в самому акті.

**Альтернативи:**

- Працювати напряму з MySQL "Моє ОСББ".
- Двосторонній sync.

**Обґрунтування:**

- Контрагент бухгалтерський (юр.особа з реквізитами на момент акту) ≠ клієнт SaaS (акаунт). Це різні життєві цикли.
- Запис в "Моє ОСББ" зменшує ризик зачепити основний продукт.
- Snapshot реквізитів у акті гарантує незмінність історичних документів при зміні даних клієнта.
- Read-only знімає питання конфліктів двостороннього sync.

**Наслідки:**

- Локальна БД (PostgreSQL або MySQL — рішення відкладене).
- Sync-механізм (раз на день, по `moeosbb_user_id IS NOT NULL`).
- `Act.client_snapshot` зберігає реквізити в JSON на момент генерації.
