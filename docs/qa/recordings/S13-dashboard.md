## Сценарій (Demo criteria § 5 S13 — стан системи за 5 секунд)

1. Логін → `/` показує три **health-banners** (ПриватБанк / Дубідок / Моє ОСББ).
   Сервіс без жодного запуску → «Ще не запускалось»; після успішного cron → ✓ з
   timestamp; після помилки (новіша за успіх) → ✗ з повідомленням і часом.
2. Три **лічильники**: «Платежів у черзі» (`in_queue`), «Платежів на апрув»
   (`awaiting_review`), «Актів очікують підпису» (`sent_to_edo`) з реальними числами.
3. Клік по лічильнику → перехід на відповідну поверхню: `/queue?tab=in_queue`,
   `/queue?tab=awaiting_review`, `/acts?status=sent_to_edo`.
4. **Ручні дії:** «Синхронізувати ПриватБанк зараз» (нова), «Синхронізувати Моє ОСББ
   зараз», «Опитати статуси Дубідок» — кожна показує короткий результат і оновлює
   банери/лічильники без повного перезавантаження.
5. **Negative:** зламати PrivatBank token → запустити ручний poll → банер ПриватБанк
   стає ✗ з повідомленням про помилку.
